>_Nexus — SSH Terminal Shell

Having trouble with >_Nexus - SSH Terminal Shell? We're here to help.

Common Issues

  • Connection refused — Verify the host, port, and username are correct. Ensure the SSH server is running and accessible from your network.
  • Authentication failed — Double-check your password or re-deploy your SSH key. If you recently changed your server password, update it in the app.
  • Host key changed warning — This means the server's fingerprint differs from what was previously stored. If you reinstalled or migrated your server, accept the new key. If unexpected, do not connect — it could indicate a security issue.
  • Connection drops frequently — This is typically a network issue. Try switching between Wi-Fi and cellular, or check if your server has an idle timeout configured.
